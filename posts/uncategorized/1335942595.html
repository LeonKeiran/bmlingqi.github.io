配置说明:

  1. demo : 指令名称
  2. description : 指令简介，说明 (接收 String 类型数据)
  3. usage : 指令用法 (接收 String 类型数据)
  4. aliases : 指令别名 (接收 List 类型数据), 可使用该项配置的所有别名代替指令名称执行

然后这个类需要实现 org.bukkit.command 包下的 CommandExecutor 接口并重写其中的 onCommand 方法

你可以在这个方法中编写执行这条命令需要做什么事情

参数说明:

  1. 第一个参数指命令执行者
  2. 第二个参数指被执行的命令类实例
  3. 第三个参数指被执行的命令名称
  4. 第四个值指命令的参数

例如:

以玩家身份执行 /demo pig 18 第一个参数则是玩家对象 (可以使用 Java 的强制转换将 CommandSender 对象转换为 Player对象 )，第四个参数则是 pig 和 18 所以第四个参数的数组的长度则是 2 ，数组第一个值 = pig 即 args[0] = pig ，数组第二个值 = 18 即 args[1] = 18

# TAB 补全

想要跟原版指令那样有提示可以继续在指令类后面添加实现 org.bukkit.command 包下的 TabExecutor 接口并重写其中的 onTabComplete 方法

  1. import org.bukkit.command.CommandExecutor;
import org.bukkit.command.TabExecutor;

/**
 * @FileName: CommandExecutor
 * @Description: Command
 * @Project: MyFirstPluginDemo
 * @Package: CommandExecutor
 * @Author: Bemil_Lingqi <bemiou_lingqi@163.com>
 * @Date 2023/1/11
 * @Website: http://www.bemiou.cn/
 * @Github: https://github.com/bmlingqi
 * @WechatPublicNumber: 贝米尔科技
 */
public class DemoCommand implements CommandExecutor, TabExecutor {
    @Override
	public boolean onCommand(CommandSender sender, Command command, String label, String[] args) {
        return false;
    }
    @Override
    public List<String> onTabComplete(CommandSender sender, Command command, String alias, String[] args) {
        return null;
    }
}

getCommand(“demo”).setTabCompleter(new DemoCommand());

该段代码会在输入 /demo 后按下空格就会弹出 test1 , test2 两个选项。在输入 /demo test1 后按下空格就会弹出 test1te , test2te 俩个选项
