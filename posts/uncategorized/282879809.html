Minecraft 插件开发 #1 环境搭建

发表于 2023-01-13
  • DTBlog
首页

# 0. 简介

  • 本教程基于 1.16.1 服务端核心开发
  • 对应 B 站视频: 传送门
  • 交流群: 791811601

# 1. 下载并安装 JDK

浏览器搜索 JDK11, 选择 orcle 官网的链接，其他也行传送门下载后双击运行傻瓜式下一步即可

# 2. 下载并安装编辑器

这里使用 Jetbrains 家的 IDEA 编辑器传送门

左侧蓝色的是付费版本只能免费只能新用户试用 30 天，右侧的是免费版，功能差不多，左侧的版本要强大点，加入交流群可获取左侧版本的激活工具

# 3. 下载服务端核心

由于本教程使用 spigot 服务端核心，所有需要下载 spigot 服务端核心，由于官网传送门没有直接提供核心下载地址而是提供了自行构建工具，所以这里给出一个核心下载地址传送门打开页面后往下滑找到对应的 Minecraft 版本那一栏，点击右侧的 Download 黄色背景按钮

在打开的页面往下滑找到下面的 You're about to download: 字样，点击字样下面的黄色的 Spigot-xxxx.jar 黄色字样即可下载核心文件

# 4. 创建项目

打开安装好的 IDEA 编辑器找到左侧 Plugins 在打开的右侧标签页中选择左侧的名为 Marketplace 选项卡如下图中上方箭头所指位置继续在下方的搜索框中输入 Chinese 然后点击回车选择下图中所指位置下方的的插件点击右侧的安装

然后安装完后重启编辑器

启动后点击创建项目

左侧选项选择最上面的第一个

右侧名称输入插件名称，语言选择 Java, 构建系统选择 Maven 或者 Gradle,JDK 选择刚刚下载的 JDK，接着展开下面的高级设置，组 ID 一般输入自己网站域名反过来写就行，例如 xxxx.com 就输入 com.xxxx 若没有，输入 github 地址也行实在不行就输入邮箱吧。工件 ID 与上面的名称输入一致即可，最后点击下面的创建

# 5. 引入依赖

打开项目跟没写的 pom.xml 文件，添加仓库地址

点击编辑器右侧的 Maven 展开生命周期，分别运行 clean 和 install

运行完后继续在 pom.xml 文件加入依赖

其中 version 节点内容可在上方添加的仓库地址的 url 里面根据下载的服务端核心版本选择输入

按照依赖项的 groupId 节点内容依次点击网页中对应的蓝色链接没有了后点击与依赖项 artifactId 对应的蓝色链接在打开的页面内选择复制与自己下载的服务端核心匹配的蓝色链接文本 (注：去掉末尾的 / 字符) 填入依赖项的 version 节点内，最后依次运行编辑器右侧的 maven 标签页内的生命周期下的 clean 和 install

# 6. 编写代码

依次展开 src 目录下的 main 目录下的 java 目录，右键 java 目录其中的包名选择 '新建'-'Java类' 在打开的窗口中输入类名 (一般为插件名称) ，新建好后，该类需要继承 org.bukkit.plugin.java 包下的 JavaPlugin 类

在类中重写 onEnable 方法。 onLoad 和 onDisable 方法可选

各方法说明:

  1. onLoad : 插件加载时执行该方法
  2. onEnable : 插件启用时执行该方法
  3. onDisable : 插件卸载时执行该方法

示例:

然后打开 src 目录下的 main 目录下的 resources 目录，右键 resources 目录新建名称为 plugin.yml 的文件，在其中输入

# 7. 创建编译工件

选择编辑器 文件-项目结构 选择左侧的 工件 点击右侧的 + 号依次选择 JAR-来自具有依赖项的模块 然后直接点击创建，

然后将右侧的 输出目录 改为当前项目跟目录，然后再目录 输出目录 后追加 \run ， run 可以自定义，点击确定。

# 8. 创建临时调试目录及运行配置

右键在项目标签页内的项目名称，选择 新建-目录 ，命名为刚刚在 项目结构 内的 输出目录 内追加的内容 (注: 去除'\')

然后将下载的服务端核心 jar 文件放进去。

点击编辑器右上方小锤子旁边的下拉框点击 编辑配置

点击左侧的 + 号选择 JAR 应用程序

右侧名称任意输入，JAR 路径选择刚刚创建的目录中服务端核心文件，程序实参可选择输入 nogui 这个表示不显示服务端默认的 GUI 界面。工作目录选择刚刚创建的目录，JRE 选择适合核心版本 java。然后往下翻找到执行前点击 + 号选择编译 Artifacts，勾选之前创建的工件。最后点击 应用-确定

# 9. 编译调试项目

点击旁边的绿色小三角运行即可
