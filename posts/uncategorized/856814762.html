# 3. 编写代码

右键包名新建一个事件类 PlayerEvents 名称随意，该类需要实现 org.bukkit.event 包下的 Listener 接口，该类没有提供方法，需要在其中创建一个方法并给方法添加 org.bukkit.event 包下的 EventHandler 注解，然后方法参数只能有一个，即是需要监听的事件类，例如: PlayerJoinEvent , AsyncPlayerChatEvent 等，列举的这两个事件分别指 玩家加入事件 ， 玩家聊天事件 更多事件可以前往传送门查看

EventHandler 注解接收两个参数:

  • priority : 指定事件的优先级 (可选，默认值为: EventPriority.NORMAL ) (接收 EventPriority 枚举类类型数据)
  • ignoreCancelled : 处理程序是否需要忽略已取消的事件 (可选，默认值为: false )(接收布尔值类类型数据)

一般情况下不需要给该注解传入参数

EventPriority 枚举的优先级顺序为: LOWEST < LOW < NORMAL < HIGH < HIGHEST < MONITOR

注:

  1. 一个类可以监听多个事件
  2. 若使用上面的方法创建的事件类在插件主类必须使用 registerEvents 方法注册事件

然后再插件主类注册这个类

最后进入游戏即可看到，加入游戏时候默认的黄色字体替换成了事件内编写的内容
